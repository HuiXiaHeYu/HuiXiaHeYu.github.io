经典目标检测

图像分类：分类器能够识别出给定图像的类别

目标检测：检测器能识别出图像中的物体，并采用边界框的形式来标记目标在图像中所处的空间位置

Two-Stage（两阶段）

模式：先提取（可能包含目标的区域），后分类（每个区域进行识别分类）

优点：效果不错

缺点：速度较慢（5fps）

深度学习之前：图像 -> 人工视觉特征（如HOG） -> 分类器（如支持向量机） -> 检测结果

2014年（Rcnn）：图像 -> 使用搜索算法从图像提取若干感兴趣区域（Rol） -> CNN分别处理每个感兴趣区域，提取特征 -> 分类器（支持向量机） -> 检测结果

Rcnn -> Fast-Rcnn -> Faster-Rcnn -> Mask-Rcnn

Mask-Rcnn：

One-Stage（单阶段）

由一个单独的网络同时完成提取候选区域（定位）和逐一识别（分类）。从而能够减少处理过程中的大量冗余操作

如：Yolo系列

优点：速度快，适合做实时监测任务

缺点：检测效果稍劣

指标解析

Iou

Intersection over Union，交集和并集的比值

越高越重合模型越好

fps

frames per second-衡量模型检测速度的每秒帧数

网络每秒可以处理多少张图像（从输入一张图像到输出图像中的目标的检测结果）

FPS的数值越大，模型的检测速度越快

mAP

评判检测效果

精度（Precision）【查准率】：检测到的样本有多少是正确的？模型挑出来的西瓜有多少比例是好瓜？

召回率（Recall）【查全率】：正确样本有多少被检测出来？所有的好瓜中有多少比例是被模型挑出来的？

置信度阈值（p-threshold）：有多少概率相信是正确的？

相关/正类无关/负类
被检测到（Retrived）true positives（TP：实际正类，被检测为正类）false positives（FP：实际负类，被检测为正类）
未被检测到false negatives（FN：实际正类，被检测为负类）true negatives（TN：实际负类，被检测为负类）

Precision=TPTP+FP=TPalldetectionsRecall=TPTP+FN=TPallgroundtruths Precision=\frac{TP}{TP+FP}=\frac{TP}{all \quad detections}\\ Recall=\frac{TP}{TP+FN}=\frac{TP}{all \quad ground \quad truths} Precision=TP+FPTP​=alldetectionsTP​Recall=TP+FNTP​=allgroundtruthsTP​

PR曲线

Precision-Recall Curve

评估分类模型在不同阈值下精确率和召回率之间的权衡关系

  • 低阈值（查的全）：低精度、高召回率

目标检测网络框架

一个常见的目标检测网络分为三大部分：

  1. 主干网络（核心）：提取图像的高级特征，减少冗余信息
  2. 颈部网络：对特征二次处理
    • 特征金字塔网络（feature pyramid network，FPN），其核心是将不同尺度的特征进行充分的融合，以提升检测器的多尺度检测的性能
    • 单独、可即插即用的模块（RFB、ASPP、Yolov4使用的SPP等）以进一步地处理和丰富特征信息，扩大模型的感受
  3. 检测头：提取类别和位置信息，输出最终的预测结果

主干网络

使用通用CNN提取特征

使用ImageNet训练的CNN作为特征提取器对接下游任务（如：目标检测、语义分割、实例分割等）：预训练模型、预训练权重【backbone】

  1. 特征全面：ImageNet是图像分类领域中最大的数据集，包含百万张自然图像，经过该数据集训练后的CNN已充分学会了如何提取“有用”的特征。
  2. 减少训练时长。

常用的主干网络模型：

  • VGG：深度
  • ResNet：残差连接，解决过深无法训练的问题
  • DarkNet
  • MobileNet：逐深度卷积（大多数轻量CNN的核心操作）。低性能移动端：如手机、无人机和其他嵌入式设备
  • ShuffleNet：通道融合，通过将每个通道的特征进行混合，弥补逐深度卷积无法使不同通道的信息进行交互的缺陷。低性能移动端

颈部网络

进一步处理特征，以适应各种任务，如目标检测、语义分割等

常用的颈部网络模型：特征金字塔、金字塔池化模块

特征金字塔

feature pyramid network-FPN，多尺度特征融合和多级检测思想

  • 浅层特征图包含更多的位置信息，且分辨率较高，感受野较小，便于检测小物体

  • 深层特征图包含更多的语义信息，且分辨率较低，感受野较大，便于检测大物体

设计自顶向下的融合方式，将深层的特征不断融合到浅层特征中，通过将浅层与深层的信息进行融合，可以有效地提升网络对不同尺度物体的检测性能

金字塔池化模块

spatial pyramid pooling-SPP，四条并行分支，连接不同大小的池化核

聚合不同尺度的显著特征，扩大网络的感受野

检测头

辅助定位和分类

RetinaNet网络：

由两条并行的分支组成，每一条分支都包含若干层普通卷积、非线性激活函数以及用于最终预测的线性卷积层

Microsoft Common Objects in Context

目标检测、图像分割、实例分割和图像标注等丰富的数据标签

具有更加贴近自然生活、场景更加复杂、目标尺度多变、光线变化显著 --> 良好的泛化性能

80类目标，11万余张图像

小目标检测如火如荼：

目标物体分别被定义为小物体、中物体和大物体这3类。小物体：像素面积小于32×32像素的目标；中物体：像素面积介于32×32至96×96像素之间的目标；大物体：其余的

  1. 大多数检测器的主干网络都会存在降采样操作，如步长为2的最大池化层或者卷积，这些降采样操作会丢失小目标的特征，这些因素都会导致网络很难充分地学习到小目标的信息
  2. 小目标的像素量远小于大目标，因此在训练网络的过程中大目标的信息会占据主导位置，也不利于网络学习小目标的信息

V1

You Only Look Once：仅使用一个卷积网络来端到端地检测目标

  • 将检测问题转化为回归问题（图像分为网格）
  • 可对视频进行实时检测

检测原理

对图像分为SxS的网格，对每一个网格求两个候选框，根据置信度（以及阈值）对候选框做微调

网络架构

为什么输入图像尺寸固定？

卷积网络结构固定的情况下，输入图像为了匹配全连接层的神经元个数（权重矩阵是一定的）

  • 架构简写：大卷积层+两个全连接+检测过程

检测过程详解

为什么是7x7x30的张量？

一共49个网格，每个网格有30值：2*边界框+置信度

[x, y, w, h, confidence, c1_prob, c2_prob, ..., cN_prob]
  • 边界框(Bounding boxes + Confidence)：中心坐标(x, y)、框的宽(w)、框的高(h)、包含目标的置信度(c)
  • 各类别的置信度(Classes)：该框属于每个类别的概率(20分类)

非极大值抑制NMS

如果多个预选框重叠？

非极大值抑制：在Iou大于一定比例的多个预选框中，按照置信度排序，取置信度最大值的预选框

缺点

  1. 重叠的物体难以检测？每个网格(cell)只能预测一个类别，如狗和猫重合到一块难搞
  2. 检测效果一般？长宽比可选但单一
  3. 小物体检测不到？先验框一般比较大
  4. 多标签难以分类？带斑点的卷毛的狗分类为：狗、斑点狗、卷毛狗

Faster R-CNN

在Fast R-CNN的基础上改进：

主要创新点：

  • 引入了区域建议网络（Region Proposal Network, RPN），这使得目标检测流程更加高效和准确。

相比之前算法的主要改进：

  1. 区域建议网络（RPN）: Faster R-CNN引入了RPN来生成候选区域。RPN是一个完全卷积的网络，它在共享的特征图上滑动一个小的卷积核，为每个位置生成多个候选区域（锚框）并预测这些区域的objectness分数和边界框回归偏移量。RPN的引入大大减少了候选区域的数量，并且使得候选区域的生成过程可以融入整个网络中，实现了端到端的训练。
  2. 锚框（Anchors）: RPN使用锚框机制来预测候选区域。锚框是一组预设的、具有不同尺度和宽高比的边界框，它们均匀地覆盖在特征图上。通过预测锚框的偏移量和objectness分数，网络可以更准确地定位目标。
  3. 非极大值抑制（Non-maximum Suppression, NMS）: Faster R-CNN在RPN之后和最终的检测阶段之前应用了非极大值抑制来去除重叠的候选区域。NMS确保每个检测目标只有一个对应的边界框，从而减少重复检测。
  4. 联合训练: Faster R-CNN允许RPN和检测网络（如Fast R-CNN）共享卷积特征并进行联合训练。这种联合训练方式有助于两个网络相互优化，提高了检测性能。
  5. 更高的检测速度和准确性: 由于RPN的引入，Faster R-CNN在保持高精度的同时，显著提高了检测速度。RPN生成的候选区域数量远少于传统的区域建议方法（如Selective Search），这使得后续的分类和边界框回归计算量大大减少。
  6. 深度特征共享: Faster R-CNN通过在RPN和Fast R-CNN之间共享深度卷积特征，减少了计算冗余，并且提高了特征的利用效率。
